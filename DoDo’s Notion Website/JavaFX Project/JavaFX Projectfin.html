What Is Cryptography?

Cryptography is securing data by Encrypting (Scrambling) it into Ciphertext (Unreadable Data) so only the intended receiver can Decrypt (Unscramble) it using a key. The data is useless without the key.

It has Two main types:

1. Symmetric Encryption

- Both sides use the Same Key for Encrypting and Decrypting data. This type is faster.

2. Asymmetric Encryption

- The sender uses receiver's Public Key to Encrypt and the receiver uses their Private Key to Decrypt the data. This type is more secure since the Decryption Key isn't shared.

What Are PBKDF2, AES-128, GCM and IV?

PBKDF2 (Password-Based Key Derivation Function 2) converts the shared password into a 128-Bit AES Key (A Key Made Out Of 128 1's & 0's) by running it through HMAC-SHA256 (Hashing Function For Converting An Input To A Fixed-Length Value) 65,536 times using a fixed Salt (A Value Added To The Password). Both users end up with the exact same key without it ever being sent over the network.

AES-128 (Advanced Encryption Standard) is the Symmetric Encryption algorithm used to scramble the message. It uses the 128-Bit Key from PBKDF2 to turn data into Ciphertext in 10 rounds.

GCM (Galois/Counter Mode) is the mode AES runs in. It adds an Authentication Tag (A Seal Made Out Of 128 1's & 0's) to all data. If even one byte changes before the data is received, the tag breaks and the message is rejected.

IV (Initialization Vector) is a random 12-Byte (Made Out Of 96 1's & 0's) value generated with SecureRandom (Java's Secure Random Number Generator) for every message. It is added to the Ciphertext as IV:Ciphertext and sent to the receiver. This makes Encrypting the same data produce differing Ciphertexts.

How My Code Functions:

1.
Regex Check: The inputs are validated using Regexes. Name must be 3-15 letters, port must be between 1024 and 65535 etc. If anything fails an error appears and nothing continues.
2.
Connecting: User1 enters a name, a port number and a password then clicks "Enter As Peer". The code opens a socket on that port and waits. User2 enters a name, the same password and User1's IP then clicks "Connect To Peer". The code connects directly using the socket with no server involved.
3.
Creating The Key: PBKDF2 With Hmac-SHA256 converts the password to a 128-bit AES Key by running it through 65,536 rounds using a Salt. Both users end up with the same exact key without it being sent over the network.
4.
Encrypting: When the user types a message and clicks "Send Message" the code generates a random 12-byte IV using SecureRandom. AES/GCM uses the key and the IV to encrypt the message and adds a 128-bit authentication tag to detect any tampering.
5.
Sending: The IV and the encrypted message are both converted to Base64 (Binary) and combined as IV:Ciphertext then sent directly over the socket.
6.
Receiving & Decrypting: The receiver's code reads the packet and splits it at the ":" into IV and Ciphertext. Both are converted back from Base64 then AES/GCM verifies the authentication tag and decrypts the message.
7.
Displaying: If the authentication tag is valid the message appears in the chat area. If the tag fails (data was tampered with) the message is rejected and an error appears instead.
